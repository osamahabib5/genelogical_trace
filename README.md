# Genealogy Ancestry Chatbot

An AI-powered Docker chatbot application for tracing African American genealogical ancestry through historical documents and records.

## Features

🔍 **Document Processing**
- Upload and process PDF, DOCX, TXT, and JSON documents
- Automatic text extraction and chunking
- Support for journals and application templates

💾 **Vector Database Storage**
- PostgreSQL with pgvector extension for semantic search
- Stores document chunks and extracted genealogical data
- Efficient similarity-based retrieval

🤖 **AI-Powered Chatbot**
- Semantic search across uploaded documents
- Natural language queries about ancestry information
- Context-aware responses with source citations
- Genealogical entity extraction (names, dates, locations, occupations, relationships)
- Responses generated by an LLM; either OpenAI's Chat API or a locally‑hosted **Ollama** model (e.g. llama3.1)

👨‍👩‍👧‍👦 **Family Tree Search**
- Search for specific people in records
- Find connected family members and relationships
- View complete genealogical connections

📊 **Web Interface**
- Modern React-based UI
- Document upload and management
- Chat interface with sources
- Family tree visualization

## Prerequisites

- Docker and Docker Compose
- OpenAI API key (optional; required only if using the OpenAI provider)
- For local inference: install and run [Ollama](https://ollama.com/) with a compatible model such as `llama3.1`
- Linux/Mac/Windows with Docker installed

## Setup Instructions

### 1. Clone and Setup

```bash
cd genealogy_traceline
cp .env.example .env
```

### 2. Configure Environment Variables

Edit `.env` and add your OpenAI API key:

```env
OPENAI_API_KEY=sk-your-api-key-here
```

Other important settings:
- `POSTGRES_USER`: PostgreSQL username (default: genealogy_user)
- `POSTGRES_PASSWORD`: PostgreSQL password (default: genealogy_password)
- `POSTGRES_DB`: Database name (default: genealogy_db)

### 3. Start the Application

```bash
docker-compose up -d
```

This will start:
- **PostgreSQL** with pgvector: `localhost:5432`
- **Backend API**: `http://localhost:8000` (configured to use the chosen LLM provider)
- **Frontend**: `http://localhost:3000`

> 💡 If you're using Ollama locally, make sure the Ollama daemon is running and reachable at the URL set in `.env` (`OLLAMA_BASE_URL`); the backend defaults to the open source llama3.1 model.

> 💡 If you're using Ollama locally, make sure the Ollama daemon is running and reachable at the URL set in `.env` (`OLLAMA_BASE_URL`); the backend defaults to the open source llama3.1 model.

### 4. Initialize Database

The database is automatically initialized when the containers start. Check logs:

```bash
docker-compose logs postgres
```

### 5. Access the Application

Open your browser and navigate to: **http://localhost:3000**

## API Endpoints

### Documents

- **POST** `/api/documents/upload` - Upload a document
  ```bash
  curl -F "file=@document.pdf" -F "document_type=journal" http://localhost:8000/api/documents/upload
  ```

- **GET** `/api/documents/list` - List all documents
- **GET** `/api/documents/{document_id}` - Get document details
- **DELETE** `/api/documents/{document_id}` - Delete a document

### Queries

- **POST** `/api/queries/search` - Search genealogical information
  ```bash
  curl -X POST -H "Content-Type: application/json" \
    -d '{"query":"Who were the ancestors of John Smith?"}' \
    http://localhost:8000/api/queries/search
  ```

- **POST** `/api/queries/ask` - Chat with the genealogy bot
  ```bash
  curl -X POST -H "Content-Type: application/json" \
    -d '{"query":"Tell me about African American soldiers in the Civil War"}' \
    http://localhost:8000/api/queries/ask
  ```

- **GET** `/api/queries/person/{name}` - Search for a specific person
- **GET** `/api/queries/family/{person_name}` - Search family connections
- **GET** `/api/queries/documents/{doc_type}` - Get documents by type
- **GET** `/api/queries/history` - Get query history

### 1. Upload a Document

1. Go to the "Upload" tab in the web interface
2. Select document type (Journal or Application)
3. Choose a PDF, DOCX, TXT, or JSON file

1. Go to the "Chat" tab
2. Enter a query like:

   - "Find all birth dates between 1850 and 1900"

### 3. Explore Family Trees

1. Go to the "Family Tree" tab
2. Enter a person's name
3. View connected family members and relationships

### 4. Manage Documents

1. Go to the "Documents" tab
2. Filter by document type
3. View document details or delete documents

## Data Schema

### Documents Table
- `id`: Document ID
- `title`: Document title
- `document_type`: 'journal' or 'application'
- `file_name`: Stored filename
- `content`: Full text content
- `upload_date`: Timestamp
- `metadata`: JSON metadata

### Document Chunks Table
- `id`: Chunk ID
- `document_id`: Reference to document
- `chunk_text`: Text content (500 chars max)
- `chunk_number`: Sequence number
- `embedding`: Vector embedding (1536 dimensions)

### Ancestry Data Table
- `id`: Record ID
- `document_id`: Reference to document
- `person_name`: Extracted person name
- `birth_date`, `death_date`: Life dates
- `birth_location`, `death_location`: Geographic info
- `occupation`: Job/profession
- `relation_type`: Family relationship
- `related_to`: Related person's name
- `embedding`: Vector embedding

## Troubleshooting

### PostgreSQL Connection Error
```bash
docker-compose logs postgres
```

Check if PostgreSQL is healthy and volumes are mounted correctly.

### OpenAI API Key Not Working
- Verify the API key format in `.env`
- Ensure the key has access to embeddings and chat models
- Check remaining API credits

### Frontend Not Loading
```bash
docker-compose logs frontend
```

Try rebuilding:
```bash
docker-compose up -d --build frontend
```

### Stop and Remove Containers
```bash
docker-compose down
docker-compose down -v  # Also remove volumes
```

## Performance Optimization

### Vector Index
The pgvector extension creates IVFFlat indexes for fast similarity search. Rebuild if needed:

```sql
REINDEX INDEX idx_chunks_embedding;
REINDEX INDEX idx_ancestry_embedding;
```

### Connection Pooling
The backend uses SQLAlchemy connection pooling. Adjust in `config.py` for more concurrent requests.

## Security Notes

- Change default PostgreSQL credentials in `.env`
- Use strong OpenAI API key
- Enable HTTPS in production
- Restrict CORS origins in `config.py`
- Validate all file uploads
